Data Access Object(DAO) API related classes - this package contains the interfaces required for implementation of the DAO pattern and criteria based searches, as well as a Reflection-based sample implementation.

The majority of the classes in this package are for the purposes of internal implementation, but the following classes are relevant to the application developer:

  • CustomReadOnlyDataSupport - for simple custom DAO implementations
  • DAO, ReadOnlyDAO - the external interface definitions
  • DAOCriteria, OrCriteria - for building query filters

Additionally, the following classes are often used from Spring, even though they aren't otherwise used directly by the application developer:

  • ReflectingDAO - the basic reflection-based default DAO implementation
  • FilterDAO - a wrapper DAO that applies a fixed filter to all queries on another DAO
  • JoinDAO - for joining two table DAOs together (like an SQL join view)
  • StarJoinDAO - for joining multiple other table DAOs together in a star formation
  • UnionDAO - for appending the results of one searches from one DAO to another's results
